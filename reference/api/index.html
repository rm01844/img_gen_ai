    🧠 AI Image Generation API¶

    This API enables developers to generate and edit images using Google Vertex AI’s Imagen models through a secure Flask REST interface.

    🔗 Base URL¶

    https://web-production-fc79.up.railway.app/

    All routes are relative to this base URL.

    🔐 Authentication¶

    All API requests require a Bearer token header:

    Authorization: Bearer

    • Each client receives a private token.
    • Tokens should never be hard-coded or shared publicly.

    Example: 

    curl -H "Authorization: Bearer your_token_here" \
     -H "Content-Type: application/json" \
     https://web-production-fc79.up.railway.app/generate

    ✨ Endpoints¶

    POST /generate¶

    Generate one or more images from a text prompt.

    Request Body

    Field Type Required Description
    prompt string ✅ Description of the image to generate
    number_of_images int ❌ Default: 1, number of images to create
    aspect_ratio string ❌ Default: "1:1", aspect ratio such as 16:9, 3:4, 9:16
    negative_prompt string ❌ Optional text describing what to avoid

    Example

    { "prompt": "A futuristic city skyline at sunset", "number_of_images": 2, "aspect_ratio": "16:9" }

    Response

    { "image_urls": [ "/static/generated_8fa3b3.png", "/static/generated_19df1d.png" ] }

    Underlying Model: imagen-4.0-generate-001 Provider: Google Vertex AI

    POST /edit¶

    Edit or transform an uploaded image using a text prompt.

    Request (multipart/form-data)

    Field Type Required Description
    image file ✅ The source image to edit
    prompt string ✅ The modification description
    number_of_images int ❌ Default: 1, number of variations to return

    Example

    curl -X POST https://web-production-fc79.up.railway.app/edit \ -H "Authorization: Bearer " \ -F "image=@castle.png" \ -F "prompt=add fog and a dragon in the sky" \ -F "number_of_images=2"

    Response

    { "image_urls": [ "/static/edited_a13c0d.png", "/static/edited_b28d9f.png" ] }

    Underlying Model: imagen-3.0-capability-001 Provider:** Google Vertex AI

    Architecture Overview¶

    graph TD
  A[Client / SDK / Postman] -->|POST /generate or /edit| B[Flask API (Railway)]
  B -->|Bearer Token| C[Auth Middleware]
  B -->|Vertex API call| D[Google Imagen Models (v3/v4)]
  D -->|Base64 images| B
  B -->|JSON response| A

    ⚠️ Error Handling¶

    HTTP Code Meaning Example Cause
    400 Bad Request Missing prompt or file
    401 Unauthorized Invalid or missing token
    404 Not Found Endpoint mismatch
    500 Internal Error Vertex API or server issue

    🧠 Notes

    • Images are saved temporarily under /static/.

    • Watermarks may be applied by Vertex AI for compliance.

    • The API auto-refreshes credentials from your GCP service account.
